Internet of Things

Estación Climatológica con Raspberry Pi y Azure IoT Hub

En este curso, se explicarán desde cero los fundamentos del Internet de las Cosas (IoT). Abarcaremos conceptos básicos, definiciones y se explicará el funcionamiento de estas tecnologías.

Además, los expositores presentarán un ejemplo práctico sobre el funcionamiento del IoT, donde fabricarán una estación climatológica utilizando el sensor de presión barométrica BMP280, la mini computadora Raspberry Pi 3 y el servicio en la nube Azure IoT Hub, para monitorear los datos entrantes, que serán capturados utilizando Python3 y la plataforma de desarrollo .NET. Finalmente, utilizando Node.js y Docker, desarrollarán una WebApp donde podremos ver los cambios de temperatura en tiempo real.

Pasos a seguir

Una vez hayamos podido capturar los datos del sensor a la Raspberry Pi, lo siguiente es envialos hacia Azure, para esto, crearemos un recurso del tipo "Azure IoT Hub".

Seleccionaremos la opción: "Gratuita". Este nivel, nos permite traabajar con 8,000 mensajes el día, suficientes para el desarrollo del proyecto.
Una vez terminada la implementación del recurso, nos dirigimos al mismo y buscamos la sección dispositivos. Dentro, selecionaremos: "Agregar DIspositivo"l. Cuando creemos un dispositivo es importante marcar la opción: "Conectar dispositivo a un Centro de IoT."
Una vez dentro del dispositivo, debemos ubicar la Cadena de Conexión Principal, se tratade una cadena de caracteres cifrada que utilizaremos para vincular los datos de un dispositivo IoT a una instancia de dispositivo en Azure IoT Hub.
Luego de conseguir nuestra cadena de conexión, el paso siguiente es exportarla a nuestra Raspberry Pi como una variable de entorno.

Para exportar una variable de entorno a nuestra Raspberry Pi, podemos utilizar el siguiente comando:

export IOT_HUB_DEVICE_CONN_STRING="<cadena_de_conexión>"

Donde la cadena de conexión es la que hemos copiado anteriormente, se escribe dentro de las comillas.

Una vez exportada, podemos comenzar a descargar las nuevas dependencias necesarias para el desarrollo del proyecto. Utilizaremos los siguientes comandos:

DEPENDENCIAS NECESARIAS UTILIZANDO PYTHON
pip install azure-iot-device pip install azure-iot-hub
DEPENDENCIAS NECESARIAS UTILIZANDO .NET
dotnet add package Microsoft.Azure.Device.Client --version 1.41.3

Para sustituir los códigos anteoriores por los nuevos, podemos descargar los archivos nuevos:

Código para Python Código para .NET

Luego de sustituir el código y ejecutar los códigos programas, podemos ver que los datos son enviados de manera satisfactoria.
